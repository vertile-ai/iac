One Source Of Truth

The user-authored source of truth is infrastructure/iac/iac.json. Terraform files are generated output under .vertile/terraform/<provider>/.

{
  "version": 1,
  "project": { "name": "example" },
  "environments": ["development", "preview", "production"],
  "providers": {
    "vercel": { "team": "example-team" },
    "aws": { "region": "us-east-1" },
    "digitalocean": {}
  },
  "apps": [{ "key": "web", "domains": ["web.example.com"] }],
  "objectStorage": [{ "key": "uploads" }],
  "databases": [{ "key": "appdb", "engine": "postgres" }],
  "queues": [{ "key": "jobs" }],
  "sandboxes": [{ "key": "runner" }],
  "clusters": [{ "key": "workers", "size": 2 }]
}

Provider Adapters

Vertile AI IaC keeps portable concepts in the manifest and compiles them into provider-specific resources. Provider overrides are available when a platform has unique behavior.

Vercel

Apps, domains, env vars, and project settings for frontend and serverless deployments.

AWS

Object storage, databases, queues, compute, and clusters for broader infrastructure needs.

DigitalOcean

Spaces, managed databases, app hosting, droplets, and pragmatic compute defaults.

Built For Small Teams And AI Workflows

Crossplane makes Kubernetes the infrastructure control plane. Vertile AI IaC uses a Git repo and CLI as the control surface, which is lighter for solo builders and app-first teams.

pnpm add -D @vertile-ai/iac
vertile-iac render --target=all --env=production
vertile-iac plan --target=aws --env=production
vertile-iac apply --target=aws --env=production --yes
Crossplane is Kubernetes-first platform infrastructure. Terraform/OpenTofu is provider-specific execution. Vertile AI IaC is app-first portable infrastructure intent.
